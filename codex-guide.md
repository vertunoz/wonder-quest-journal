# Инструкция к запросам в Codex

## 0. Разовая настройка
1. Положи `AGENTS.md` в корень репозитория, `contracts.md` и папку `parts/` рядом.
2. Заполни в `AGENTS.md` блоки «Стек» и «Команды» (особенно команду тестов —
   Codex прогоняет её перед завершением задачи).
3. Запускай сильнейшую модель для кода (на 2026 — линейка GPT-5.5; проверь текущий
   дефолт). Codex сам подбирает усилие рассуждений и умеет сжимать длинные сессии.

## 1. Главный принцип контекста
В сессию НЕ вставляй весь план. Контракты типов уже в `AGENTS.md` (читается
автоматически). В запросе ты ссылаешься на ОДИН файл из `parts/`. Всё.

## 2. Шаблон запроса (4 блока)
```
[КОНТЕКСТ]  Строим Part 4 (Vision Board). Спека: parts/part-04-vision-board.md.
            Контракты типов — в AGENTS.md, свои поля не выдумывай.
[ЗАДАЧА]    Реализуй 4a: CRUD бордов и addToBoard по контракту Board/BoardItem.
[ГРАНИЦЫ]   Не трогай decision-движок и квиз. Схему данных не меняй.
[ГOTOВО]    Сначала падающие тесты на CRUD и addToBoard, затем реализация до
            зелёного. Тесты не менять. Прогони команду тестов перед завершением.
```

## 3. Протокол на каждую часть
1. Сложная часть (6, 9) → сперва: «Составь план реализации Part N. Код не пиши,
   жди моего ок.» Проверь план → «ок, реализуй».
2. Простая часть → сразу шаблон из п.2.
3. Если часть упёрлась в контекст — режь по контракту (Part 4 → 4a CRUD, 4b раскладка).
4. После работы — `/review` или ревью diff; правки скармливай как контекст следующего хода.

## 4. Порядок
0 (contracts) → 1 → 2 → 3 → 4 → 5  (это рабочий MVP),  далее V2 (6–9), V3 (10–12)
в любом порядке внутри версии. Каждая часть — отдельная сессия/ветка.

## 5. Что НЕ делать
- Не вставлять product-plan.docx в промпт.
- Не просить «сделай всё сразу» — один вертикальный срез за раз.
- Не давать менять тесты и схему данных без явного запроса.

## 6. Опционально (продвинутое)
Повторяющиеся части можно превратить в Codex Skill (`SKILL.md` в `~/.agents/skills/`)
с описанием-триггером — тогда Codex подхватывает паттерн сам. Команды тестов в
AGENTS.md = твой Definition of done на уровне репозитория.

## 7. Готовые однострочники-инвокации
- `Part 1: parts/part-01-place-card.md — собери карточку по шаблону из codex-guide.md.`
- `Part 2: parts/part-02-quiz.md — квиз → TasteProfile, test-first.`
- `Part 3: parts/part-03-decision-engine.md — чистая computeVerdict + табличные тесты.`
- `Part 4a: parts/part-04-vision-board.md — только CRUD борда + addToBoard.`
- `Part 5: parts/part-05-share-clone.md — публичная страница + cloneBoard.`
- (далее по списку 6–12 аналогично)
